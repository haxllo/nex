**Part A — Add Tantivy memory query (tantivy_search.rs):**

Add a method to query Tantivy's arena memory usage:
```rust
/// Return approximate memory used by the Tantivy writer arena and index structures.
/// This is the 16MB arena plus any segment caches held by the reader.
pub fn mem_usage_bytes(&self) -> usize {
    // The IndexWriter arena size is the parameter we passed to index.writer(16_000_000).
    // Tantivy doesn't expose current arena usage directly, but we can approximate:
    // - Writer arena: 16MB (if active) or 0 (if dropped)
    // - Reader segment cache: searcher memory (estimate from num_docs * avg doc size)
    let writer_arena = 16_000_000; // arena size from index.writer() call
    let reader = self.reader.searcher();
    let segment_mem = reader.segment_readers().len() * 2_000_000; // rough 2MB per segment
    writer_arena + segment_mem
}
```

Actually, keep it simple. The primary metric we want is process-level memory. Tantivy's internal memory is a component. For now, just return the arena size constant + rough segment estimate. We can refine later with actual WPR profiling.

**Part B — Add memory logging to CoreService (core_service.rs):**

Add `pub(crate) fn log_memory_stats(&self)` that uses Windows `GetProcessMemoryInfo` to get:
- Working set size (bytes)
- Private bytes (via `K32GetProcessMemoryInfo` or `GetProcessMemoryInfo`)

Implementation using `windows-sys` (already a dependency):
```rust
#[cfg(target_os = "windows")]
pub(crate) fn log_memory_stats(&self) {
    use windows_sys::Win32::System::ProcessStatus::{GetProcessMemoryInfo, PROCESS_MEMORY_COUNTERS};
    use windows_sys::Win32::System::Threading::GetCurrentProcess;
    
    let mut pmc: PROCESS_MEMORY_COUNTERS = unsafe { std::mem::zeroed() };
    pmc.cb = std::mem::size_of::<PROCESS_MEMORY_COUNTERS>() as u32;
    let ret = unsafe { GetProcessMemoryInfo(GetCurrentProcess(), &mut pmc, pmc.cb) };
    if ret != 0 {
        let working_set_mb = pmc.WorkingSetSize / (1024 * 1024);
        let pagefile_mb = pmc.PagefileUsage / (1024 * 1024);
        
        let tantivy_mem = match self.tantivy_index.lock() {
            Ok(g) => g.as_ref().map_or(0, |idx| idx.mem_usage_bytes()),
            Err(_) => 0,
        } / (1024 * 1024);
        
        let icon_cache_entries = 0; // read from IconCache if accessible, else 0
        
        crate::logging::info(&format!(
            "[nex] memory_stats working_set_mb={} pagefile_mb={} tantivy_mb={} icon_entries={}",
            working_set_mb, pagefile_mb, tantivy_mem, icon_cache_entries
        ));
    }
}

#[cfg(not(target_os = "windows"))]
pub(crate) fn log_memory_stats(&self) {
    // No-op on non-Windows
}
```

**Part C — Trigger from runtime_loop (runtime_loop.rs):**

In `run_windows_runtime()` or wherever the main event loop runs, add a periodic check every 30 seconds. There should already be a timer mechanism — find where idle events are processed (like `prune_stale_items_if_due`) and add a `log_memory_stats` call with a 30-second interval.

Add `last_memory_log: Instant` field to the runtime state. In the main loop, check if 30s elapsed since last log, call `service.log_memory_stats()`, update timestamp.

**DO NOT block the main thread** — the `GetProcessMemoryInfo` call is fast (microsecond-scale), but keep it behind the interval check to avoid spamming logs.

**Keep non-Windows compatibility:** Use `#[cfg(target_os = "windows")]` guards. The method always exists but is a no-op on non-Windows.
