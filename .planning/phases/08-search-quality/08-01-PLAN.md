<objective>
Implement app deduplication so the same application discovered from multiple sources (Start Menu .lnk, uninstall registry entry, raw .exe path) appears only once in search results. Currently, searching for "firefox" might show 3-4 entries for Firefox — this plan ensures only the highest-scored variant is shown.

Purpose: Eliminate result clutter where 8 variants of the same app push genuinely different results off the visible list. This directly addresses R3.3's "Result diversity (don't show 8 variants of the same app)" requirement from the ROADMAP.

Output: `deduplicate_apps()` function integrated into the search ranking pipeline, with tests proving same-app variants collapse to a single result.
</objective>

Add an app-deduplication post-processing step to `search_with_filter_with_boosts()`. The deduplication runs AFTER scoring and sorting but BEFORE the final `take(limit)`.

**Algorithm:**

Create a new function `fn deduplicate_apps(scored: &mut Vec<ScoredItem<'_>>)` that:

1. Groups items by a dedup key: `(normalize_for_search(&item.title), basename(&item.path))`
   - `basename()` extracts the filename portion: `path.rsplit('\\').next().unwrap_or(path)` — if it has an extension, strip it: `basename.rsplitn(2, '.').last().unwrap_or(basename)`
   - Only group items where `item.kind.eq_ignore_ascii_case("app")` — non-apps pass through untouched
   - The grouping key = `(normalized_title, normalized_basename)`

2. For each group with > 1 item, keep ONLY the item with the highest `score`
   - Remove lower-scored items from the Vec
   - This is a stable retention: the `ScoredItem` with the highest score stays at its original position

3. After dedup, the Vec is smaller. The `take(limit)` at the end of `search_with_filter_with_boosts()` naturally fills the vacated slots with the next-best non-duplicate items.

**Implementation details for deduplicate_apps:**

```rust
fn same_base_name(a: &str, b: &str) -> bool {
    let base_a = extract_base_name(a);
    let base_b = extract_base_name(b);
    base_a == base_b
}

fn extract_base_name(path: &str) -> String {
    let filename = path.rsplit('\\').next().unwrap_or(path);
    // Strip extension (e.g., "firefox.exe" → "firefox")
    let stem = if let Some(pos) = filename.rfind('.') {
        &filename[..pos]
    } else {
        filename
    };
    normalize_for_search(stem)
}

fn deduplicate_apps(scored: &mut Vec<ScoredItem<'_>>) {
    use std::collections::HashMap;
    
    let mut groups: HashMap<(String, String), Vec<usize>> = HashMap::new();
    
    // Group app items by (normalized_title, normalized_basename)
    for (idx, s) in scored.iter().enumerate() {
        if !s.item.kind.eq_ignore_ascii_case("app") {
            continue;
        }
        let key = (
            normalize_for_search(&s.item.title),
            extract_base_name(&s.item.path),
        );
        groups.entry(key).or_default().push(idx);
    }
    
    // For each group, find the index with the highest score, mark others for removal
    let mut remove_indices: Vec<usize> = Vec::new();
    for (_, indices) in groups.iter() {
        if indices.len() <= 1 {
            continue;
        }
        // Find the best index (highest score, then best match_kind)
        let best_idx = indices
            .iter()
            .max_by(|&&a, &&b| compare_scored(&scored[a], &scored[b]))
            .copied()
            .unwrap();
        for &idx in indices {
            if idx != best_idx {
                remove_indices.push(idx);
            }
        }
    }
    
    // Sort descending so we remove from back to front (preserves earlier indices)
    remove_indices.sort_unstable_by(|a, b| b.cmp(a));
    for idx in remove_indices {
        scored.remove(idx);
    }
}
```

**Edge cases:**
- Items with empty paths: extract_base_name returns empty string — groups only with other empty-path items
- Items with no extension: filename used as-is for basename comparison
- Paths with forward slashes: `rsplit('\\')` won't find them — handle by also checking for `/`: use `path.rsplit(['\\', '/']).next()`
- Ties in score: `compare_scored` already provides a total order (score → match_kind → source_rank → title_len → title → id) so `max_by` is deterministic

**IMPORTANT — Keep non-app items unaffected:** The `kind.eq_ignore_ascii_case("app")` check ensures only apps are deduplicated. Files, folders, actions, and clipboard items pass through unmodified.

#[test]
fn dedup_removes_same_app_from_different_sources() {
    // Firefox discovered from 3 sources: Start Menu .lnk, uninstall registry, raw .exe
    let items = vec![
        SearchItem::new("firefox-lnk", "app", "Firefox",
            "C:\\Users\\Admin\\AppData\\Roaming\\Microsoft\\Windows\\Start Menu\\Programs\\Firefox.lnk"),
        SearchItem::new("firefox-uninstall", "app", "Firefox",
            "C:\\Program Files\\Mozilla Firefox\\uninstall\\helper.exe"),
        SearchItem::new("firefox-exe", "app", "Firefox",
            "C:\\Program Files\\Mozilla Firefox\\firefox.exe")
            .with_usage(50, 2_000_000_000), // heavily used variant
        SearchItem::new("chrome", "app", "Chrome",
            "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe"),
    ];

    let results = nex_core::search::search(&items, "fir", 10);
    let ids: Vec<&str> = results.iter().map(|i| i.id.as_str()).collect();
    
    // Only one Firefox should appear (the highest-scored: firefox-exe with high usage)
    assert!(ids.contains(&"firefox-exe"));
    assert!(!ids.contains(&"firefox-lnk"));
    assert!(!ids.contains(&"firefox-uninstall"));
    // Chrome should still appear
    assert!(ids.contains(&"chrome"));
    // 1 Firefox + 1 Chrome = 2 results
    assert_eq!(results.len(), 2);
}

#[test]
fn dedup_fills_slots_with_next_best_non_duplicates() {
    // 8 variants of Firefox + 1 Chrome. limit=5. After dedup: 1 Firefox + 1 Chrome = 2 results
    // But actually: the limit is 5, and we have 9 items → after dedup we have 2 items.
    // The dedup removes items from the sorted Vec, so the remaining non-duplicate items
    // naturally fill up to the limit. Since we only have 2 unique items, only 2 returned.
    let mut items = Vec::new();
    for i in 0..8 {
        items.push(SearchItem::new(
            &format!("firefox-{}", i), "app", "Firefox",
            &format!("C:\\Program Files\\Firefox{}\\firefox.exe", i),
        ));
    }
    items.push(SearchItem::new("chrome", "app", "Chrome",
        "C:\\Program Files\\Google\\Chrome\\Application\\chrome.exe"));

    let results = nex_core::search::search(&items, "fire", 5);
    
    // Only 1 Firefox + Chrome = 2 results (not 5, since only 2 unique items match)
    let firefox_count = results.iter().filter(|i| i.title == "Firefox").count();
    assert_eq!(firefox_count, 1);
    assert!(results.iter().any(|i| i.id == "chrome"));
    assert_eq!(results.len(), 2);
}

#[test]
fn dedup_does_not_affect_non_app_items() {
    let items = vec![
        SearchItem::new("firefox-app1", "app", "Firefox",
            "C:\\Program Files\\Firefox\\firefox.exe"),
        SearchItem::new("firefox-app2", "app", "Firefox",
            "C:\\Users\\Admin\\Desktop\\Firefox.lnk"),
        SearchItem::new("firefox-file", "file", "Firefox",
            "C:\\Docs\\Firefox.txt"),
        SearchItem::new("firefox-folder", "folder", "Firefox",
            "C:\\Projects\\Firefox"),
    ];

    let results = nex_core::search::search(&items, "firefox", 10);
    let ids: Vec<&str> = results.iter().map(|i| i.id.as_str()).collect();
    
    // App dedup: only 1 of the 2 app items survives
    let app_count = results.iter().filter(|i| i.kind == "app" && i.title == "Firefox").count();
    assert_eq!(app_count, 1);
    // File and folder are NOT deduplicated
    assert!(ids.contains(&"firefox-file"));
    assert!(ids.contains(&"firefox-folder"));
    // Total: 1 app + 1 file + 1 folder = 3 results
    assert_eq!(results.len(), 3);
}
