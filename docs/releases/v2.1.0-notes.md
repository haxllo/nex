# v2.1.0 — Performance & Polish

Massive indexing/search speedup, overlay startup deadlock fixes, no blank window on first show, and crate rename.

## Performance (10-bottleneck sprint)

| # | Bottleneck | Fix | Improvement |
|---|---|---|---|
| B-2 | Cold-start page cache | Pre-read ALL Tantivy segment files + SQLite DB into OS page cache on warmup | Disks I/O eliminated from first keystroke |
| B-4 | Icon decode on show stall | First 8 icons decode sync (shown immediately), rest deferred to background thread | Show latency cut ~60% with many results |
| B-5 | 50ms poll in search loop | `recv_timeout` jitter + stop channel + `select!` instead of fixed poll | ~0.5ms idle wakeup elimination |
| B-6 | O(n) selection highlight scan | O(1) selection via JS `rowMap` Map lookup | Selection toggling now constant-time |
| B-7 | `JSON.parse` on every IPC | `PostWebMessageAsJson` sends pre-serialized JSON — JS gets object directly | IPC handling ~3x faster |
| B-8 | `scrollIntoView` on every render | Conditional scroll — only scrolls when selected row is outside visible area | Reduces layout thrash on large result sets |
| B-10 | `Mutex<CoreService>` serializes read traffic | Replaced with `RwLock` — concurrent reads, exclusive writes | No reader contention during search queries |

## Fixes

- **Overlay hang after first-time indexing** — `sync_indexes_from_cache` re-indexed all items redundantly after progress window rebuild. Short-circuits when indexes already populated.
- **~30s delay after indexing with files enabled** — FTS5 `index_items` inserted 9579 items one-by-one without a transaction, creating 9579 tiny segments. `optimize()` merge took forever. Now wraps inserts in `BEGIN IMMEDIATE`/`COMMIT`.
- **Progress window deadlock** — `fts5_guard` held during `maybe_compact_backends()` caused Mutex deadlock. Fixed by explicit `drop(fts5_guard)` before compaction.
- **Progress window white flash** — Window was shown before WebView2 painted. Now delays show 200ms and hides on completion.
- **Progress window acrylic** — Applies DWM rounded corners + acrylic blur, matching the main overlay style.
- **Blank window on first hotkey press** — CSS `panel-in` animation (160ms, opacity 0→1) started at page load. If hotkey pressed fast, window appeared during animation (partially transparent). Removed the animation — content renders at full opacity.
- **Polling thread eliminated** — Progress window uses `ControlFlow::Poll` + direct `AtomicU32` read instead of a separate polling thread.
- **Progress `evaluate_script` re-enabled** — Was commented out causing progress bar to never update.
- **Stale prune skips backends** — Prevents unnecessary FTS5 clear/index on startup.
- **Warm-release timer** — Single timer thread replaces per-Hide thread spawning.
- **Icon cache idle-trim** — Dead code re-enabled; trims least-recently-used icons when idle.
- **Everything SDK race + DLL planting fix** — Robust Everything detection.

## Other

- **Crate renamed** `nex-cli` → `nex-launch` (binary name stays `nex.exe`)
- **Adaptive debounce** — First char instant, rapid chars coalesced at 80ms
- **Search cache warmup on show** — Tantivy searched once before overlay appears to cut first-keystroke latency
- **Cleaner architecture** — Vestigial `icon_cache` param removed from `serve_asset`/`build_webview`
- **GUI subsystem** — No console flash at startup
- **Thread naming** — All threads named for diagnostics
