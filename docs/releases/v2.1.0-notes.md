# v2.1.0 — Performance & Polish

Massive indexing/search speedup, overlay startup deadlock fixes, no blank window on first show, and crate rename.

## Changelog (24 commits since v2.0.0)

### Performance

- **B-2**: Pre-read ALL Tantivy segment files + SQLite DB into OS page cache — disk I/O eliminated from first keystroke
- **B-4**: Icon decode throttle — first 8 sync, rest deferred to background thread — show latency cut ~60%
- **B-5**: `recv_timeout` jitter + stop channel + `select!` instead of 50ms fixed poll — ~0.5ms idle wakeup eliminated
- **B-6**: O(1) selection highlight via JS `rowMap` Map lookup — constant-time toggling
- **B-7**: `PostWebMessageAsJson` sends pre-serialized JSON, JS gets object directly — IPC ~3x faster
- **B-8**: Conditional `scrollIntoView` only when selected row outside visible area — less layout thrash
- **B-10**: `RwLock` replaces `Mutex<CoreService>` — concurrent reads, no reader contention
- **Search cache warmup** on overlay show — Tantivy searched once before first keystroke
- **Stale prune** moved off search critical path to background thread
- **Adaptive debounce** — first char instant, rapid chars coalesced at 80ms
- **Blocking `evaluate_script`** replaced with fire-and-forget `PostWebMessageAsString`

### Bug Fixes

- **Overlay hang after first indexing** — `sync_indexes_from_cache` short-circuits when indexes already populated
- **~30s delay after indexing with files** — FTS5 `index_items` wraps inserts in `BEGIN IMMEDIATE`/`COMMIT` (was 9579 tiny segments)
- **Progress window deadlock** — `drop(fts5_guard)` before `maybe_compact_backends()`
- **Progress window white flash** — Delays show 200ms until WebView2 paints
- **Progress window acrylic** — DWM rounded corners + acrylic blur
- **Progress `evaluate_script` re-enabled** — Progress bar now actually updates
- **Blank window on first hotkey** — Removed `panel-in` CSS animation (160ms opacity fade caused blank on fast press)
- **Polling thread eliminated** — `ControlFlow::Poll` + direct `AtomicU32` read replaces separate thread
- **Warm-release timer** — Single timer thread replaces per-Hide thread spawning
- **Icon cache idle-trim** — Dead code re-enabled, trims LRU icons when idle
- **Everything SDK race + DLL planting** — Robust Everything detection

### Architecture

- **Crate renamed** `nex-cli` → `nex-launch` (binary stays `nex.exe`)
- **GUI subsystem** — No console flash at startup
- **Cleanup**: Remove vestigial `icon_cache` param from `serve_asset`/`build_webview`
- **Thread naming** — All threads named for diagnostics
- **Dead SVG icon deps removed**
