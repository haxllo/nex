Fix shutdown hang where the process lingered in Task Manager after quitting from system tray until the hotkey was pressed once. Root cause was `std::sync::OnceLock::clone()` creating an independent deep copy in Rust 1.96, so the thread ID set by the worker was never visible to `Drop`, and the hotkey listener thread never received `WM_QUIT`.

## Changelog (7 commits since v2.1.0)

### Bug Fixes

- **shutdown hang**: `OnceLock::clone()` deep copy hid thread ID from `Drop` — listener thread never posted `WM_QUIT`, blocking `handle.join()` forever. Wrapped in `Arc` to share state.
- **release notes**: removed duplicate title heading.
