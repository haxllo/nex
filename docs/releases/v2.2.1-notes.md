Fixes the warm-release hotkey miss when the WebView2 process has been fully torn down by the idle timer.

## Changelog (1 commit since v2.2.0)

### Bug Fixes

- **First hotkey after warm-release teardown now works**: When the warm-release timer drops the WebView after a period of inactivity, pressing the hotkey rebuilds it. During `build_webview()`, Windows sends spurious `Focused(true)`/`Focused(false)` events that triggered the click-outside-to-dismiss handler, hiding the overlay before the page could paint. The second hotkey press worked because the WebView already existed and no focus events fired. Added `!show_pending` guard to the escape condition — the overlay now stays open during the build phase.
