Fixes startup registration — Nex now correctly writes the Windows Run key so it appears in Task Manager's startup apps list.

## Changelog (2 commits since v2.2.1)

### Bug Fixes

- **Nex not showing in Task Manager startup apps**: `options.background` defaulted to `true`, which caused the argument parser to reject any CLI command (`--set-launch-at-startup`, `--ensure-config`, `--quit`) with *"background mode is only valid with normal run mode"* — an invisible failure because `windows_subsystem = "windows"` suppresses the console. The Run key was never written. Fixed by automatically setting `background = false` for all non-`Run` commands.
