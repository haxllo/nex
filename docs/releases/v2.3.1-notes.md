Resolves all 20 robustness audit issues plus 3 critical shutdown bugs — tray Quit now exits cleanly, arrow key navigation is ~20x faster, and startup code is hardened against poisoned locks.

## Changelog (30 commits since v2.3.0)

### Bug Fixes

- **Shutdown hang**: fixed 3-part root cause preventing process exit after tray Quit — COM STA deadlock resolved by switching to MTA, `FileWatcherHandle::drop` made non-blocking with `mem::forget`, `std::process::exit(0)` added as safety net
- **COM init per invocation**: moved `CoInitializeEx` to `thread_local!` for the persistent prefetch thread
- **Startup unwrap hardened**: replaced 5 bare `.unwrap()` calls in `runtime_loop.rs` with `unwrap_or_else` for poisoned lock recovery
- **Escape handler duplicate**: fixed pre-existing bug where `search_session.clear()`, `search_worker.clear_session()`, and `try_recv` drain were duplicated with inconsistent indentation
- **Indexer thread panic**: indexer panic now handled gracefully instead of crashing main thread via `.expect()` on poisoned result slot
- **Stale pruner shutdown**: added `AtomicBool` stop signal so the pruner thread exits cleanly on shutdown
- **File watcher shutdown**: `stop_file_watchers()` called before worker thread join to prevent deadlock
- **Runtime worker panic guard**: wrapped `worker.run()` in `catch_unwind` to prevent silent UI freeze
- **Blocking search reads**: replaced blocking `cached_items.read()` with `try_read()` in search fallback paths
- **Icon prefetch thread accumulation**: replaced per-call thread spawning with single persistent `nex-icon-prefetch` thread
- **last_touch unbounded growth**: added `clean_orphaned_touches()` to prevent `HashMap` from growing beyond LRU capacity

### Performance

- **Arrow key navigation**: `UiCommand::SelectChanged` sends only `{"selected": idx}` (~20 bytes) instead of full state snapshot (~134KB); arrow keys now ~1ms instead of ~20ms
- **Icon delivery**: dual `PostWebMessageAsJson` — lightweight state JSON first, icon data JSON second; state lock hold reduced from ~5ms to ~0.1ms
- **Lock contention**: `push_state()` clones `ShimState` first, drops lock, then serializes outside lock
- **CSS compositor**: removed redundant `backdrop-filter: saturate(140%)` that doubled compositor work
- **Row animation**: moved `@keyframes row-in` to `#body.initial-render .row` to prevent 20 concurrent animations per keystroke
- **JS debounce**: reduced from 80ms to 40ms for faster first keystroke response
- **Painted IPC**: `post("painted")` only sent after `show_pending = true`, eliminating unnecessary IPC during typing
- **Personalization cache**: 5-second TTL cache for SQLite personalization queries, eliminating ~1-2ms per search
- **Tantivy incremental sync**: writer lock held only during fast write phase (~1ms) instead of full scan (~100ms)

### Architecture

- **Persistent prefetch thread**: single `nex-icon-prefetch` thread with shared work slot replaces per-call thread spawning
- **COM MTA**: icon prefetch uses `COINIT_MULTITHREADED` instead of `COINIT_APARTMENTTHREADED` to prevent `ExitProcess` deadlock
- **catch_unwind on search worker**: prevents search panic from killing the worker thread

### Documentation

- New `docs/plans/robustness-audit.md` with all 20 issues tracked and resolved
- Updated `docs/plans/audit-and-roadmap.md` for WebView2 overlay
- Updated `AGENTS.md` overlay architecture section
- Updated `docs/v2-verified-gaps.md` with WebView2 status
