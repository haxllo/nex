Fixed open/close overlay flash and improved rendering reliability.

## Changelog (6 commits since v2.4.1)

### Bug Fixes

- **White flash on open**: Quick launch items now load into state before `UiCommand::Show` is posted, so the initial state snapshot includes rows. Previously a race allowed the event loop to process `Show` before the DB read completed, causing the window to appear empty then populate a frame later.
- **Pulse on open**: `post("resize")` now fires before `post("painted")`, so Rust sets the window height BEFORE making it visible (instead of at INITIAL_HEIGHT=60px then jumping).
- **Opacity flash on open**: Removed opacity 0→1 from `row-in` animation. Rows now start fully visible with only a subtle translateY slide.
- **Body flash on close**: `UiCommand::Hide` now hides the window BEFORE pushing cleared state, so the user never sees the empty body render.
- **Divider visible on idle**: Added `class="idle"` to `<div id="body">` in HTML so divider + list area are hidden by default until rows render.
