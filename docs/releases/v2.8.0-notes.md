Overlay flash fixes, GPU layer preservation, themed icons, and web search on no-results.

## Changelog (10 commits since v2.7.0)

### Features

- **Web search on no-results** — empty search results now show a "Search Web for [query]" action row that opens your configured search provider in browser.
- **Themed placeholder icons** — app placeholder icon respects theme: dark icon on light theme, light icon on dark theme.
- **Themed web search icon** — web search row gets a themed globe icon.

### Bug Fixes

- **DWM hide flash** — `DWMWA_TRANSITIONS_FORCEDISABLED` disables DWM window transition animation, preventing white flash on hide.
- **Focus delay** — `focus_input` moved to `Painted` handler; first paint no longer blocked by JS eval.
- **Redundant `.selected`** — duplicate `.selected` re-apply removed from `render()`.
- **GPU layer preservation** — `#body.idle` uses `visibility: hidden` + `translateZ(0)` instead of `display: none`, keeping the GPU compositor layer alive across idle→active transitions.
- **Overlay resize** — `resized_this_cycle` removed; resize always processed, window follows panel in both directions.
- **Max window height** — fixed max-height constraint to prevent overlay exceeding screen bounds.

### Architecture

- **Acrylic tint** — darkened from `(18,18,20,130)` to `(0,0,0,230)` (90% opaque black) for cleaner appearance.
- **WebView2 background** — set to transparent black `(0,0,0,0)` to avoid white flash during WebView init.
